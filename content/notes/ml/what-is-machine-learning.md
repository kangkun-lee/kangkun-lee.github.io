# $Machine$ $Learning$ $이란$

## Machine Learning의 개념

머신러닝(Machine Learning)은 **명시적으로 규칙을 다 코딩하지 않아도**, 데이터를 통해 **패턴(규칙)을 학습**해서 예측/분류/추천 같은 일을 하도록 만드는 방법이다.

예를 들어,
- “스팸 메일은 이런 단어가 많으면 스팸” 같은 규칙을 사람이 하나하나 만들기보다는,
- 스팸/정상 메일 데이터를 많이 보여주고 **스스로 기준을 찾게** 만드는 방식이다.

즉, 머신러닝은 이런 흐름으로 이해하면 된다.

1) 데이터 수집  
2) 특징(feature) 만들기  
3) 모델(model)이 패턴 학습  
4) 새로운 데이터에 대해 예측/판단

---

### Machine + Learning

#### $Machine$
여기서 Machine은 “기계”라기보다, **학습을 수행하는 시스템**(프로그램/모델)을 말한다.

- 입력: 데이터 ($x$)
- 출력: 예측값 ($\hat{y}$)
- 목표: ($\hat{y}$)가 진짜 ($y$)에 가깝게 만들기

즉, “입력 넣으면 출력 뱉는 함수”를 만드는 과정이다.

#### $Learning$
Learning은 “공부한다”는 뜻 그대로,
**정답 데이터를 보고 틀린 정도를 줄이는 방향으로 모델을 업데이트**하는 걸 의미한다.

대표적으로는 이런 형태다.

- 손실함수(loss): “얼마나 틀렸는지 점수”
- 최적화(optimization): 그 점수를 최소로 만드는 과정

결국 머신러닝의 본질은:

> “손실을 줄이도록 파라미터를 조정하는 최적화 과정"

이라고 보면 된다.

## Machine Learning의 종류

머신러닝은 보통 **학습 데이터에 정답(label)이 있는지**로 분류한다.

### 1) 지도학습(Supervised Learning)
정답이 있는 데이터를 사용한다.

- 입력 ($x$) → 정답 ($y$) 가 같이 있음
- 예: 스팸 분류(스팸/정상), 집값 예측(가격)

대표 태스크:
- **분류(Classification)**: 결과가 카테고리(스팸/정상)
- **회귀(Regression)**: 결과가 연속값(집값/온도/매출)

### 2) 비지도학습(Unsupervised Learning)
정답이 없다. 데이터 자체의 구조를 찾는다.

- 예: 비슷한 고객끼리 묶기(군집화), 차원 축소(PCA)

### 3) 강화학습(Reinforcement Learning)
정답 대신 **보상(reward)** 을 통해 학습한다.

- 예: 게임 AI, 로봇 제어
- “지금 선택이 좋은가?”를 보상으로 평가받고 개선

## Machine Learning의 활용

머신러닝은 “예측”만 하는 게 아니라, **의사결정을 자동화** 하는 데 널리 쓰인다.

### 1) 추천 시스템
- 유튜브/넷플릭스/쇼핑몰 추천
- “내가 좋아할 확률”을 예측해서 순서를 정함

### 2) 자연어 처리(NLP)
- 번역, 요약, 검색, 챗봇
- 문장을 숫자 벡터로 바꿔서 모델이 이해하도록 함

### 3) 컴퓨터 비전(CV)
- 얼굴 인식, 객체 탐지, 의료 영상 분석
- 이미지를 보고 “무엇이 있는지” 판단

### 4) 금융/산업 예측
- 주가/수요/재고 예측, 이상 탐지
- 사기 거래 탐지(Anomaly Detection)

## Machine Learning의 발전

머신러닝이 대중화된 큰 이유는 **데이터·연산·알고리즘** 세 가지가 같이 발전했기 때문이다.

### 1) 데이터의 폭발
- 인터넷/모바일로 데이터가 엄청나게 쌓임
- “학습할 재료”가 많아짐

### 2) GPU와 연산 자원
- 딥러닝은 연산량이 큰데, GPU가 이를 가능하게 함
- 클라우드로 누구나 대규모 학습 가능

### 3) 알고리즘과 모델 구조의 발전
- 전통 ML(선형 회귀, SVM, 트리) → 딥러닝(신경망)
- 특히 Transformer 이후, 언어/이미지/음성 전반에서 성능이 크게 향상됨
