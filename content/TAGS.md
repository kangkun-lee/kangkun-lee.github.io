# 태그 가이드

블로그에서 사용하는 태그를 정리한 문서입니다.

## 기술 스택

- `Python` - Python 프로그래밍
- `JavaScript` - JavaScript/TypeScript
- `React` - React 프레임워크
- `Vue` - Vue.js
- `Node.js` - Node.js
- `Docker` - Docker 컨테이너
- `Kubernetes` - K8s
- `AWS` - Amazon Web Services
- `Hugo` - Hugo 정적 사이트 생성기

## 주제/분야

- `머신러닝` - 머신러닝 관련
- `딥러닝` - 딥러닝 관련
- `웹개발` - 웹 개발 전반
- `알고리즘` - 알고리즘 문제 해결
- `데이터베이스` - DB 관련
- `DevOps` - DevOps 관련
- `통계` - 통계학
- `데이터분석` - 데이터 분석

## 포스트 유형

- `튜토리얼` - 단계별 가이드
- `팁` - 유용한 팁
- `이론` - 이론 설명
- `리뷰` - 제품/서비스 리뷰
- `프로젝트` - 프로젝트 소개
- `회고` - 회고록

## 난이도

- `초급` - 초보자 대상
- `중급` - 중급자 대상
- `고급` - 고급자 대상

## 사용 규칙

1. **일관성**: 같은 의미의 태그는 통일 (예: `ML` 대신 `머신러닝`)
2. **대소문자**: 기술명은 대문자로 시작 (예: `Python`, `React`)
3. **한글/영문**: 주제는 한글, 기술명은 영문 권장
4. **개수**: 포스트당 3-7개 정도가 적당
